## 🐛 常见问题排查

### 问题1: 前端显示旧格式（DeepAnalysisCard）

**症状**: 看到L1-L5分析卡片，而不是4条使命

**排查步骤**:
1. 检查后端日志是否有 `[v7.302]` 标记
2. 检查后端是否输出4条使命格式
3. 刷新浏览器页面（Ctrl+F5 强制刷新）

**解决方案**:
```bash
# 重启后端服务
cd intelligent_project_analyzer
# Ctrl+C 停止服务
python -m uvicorn main:app --reload
```

---

### 问题2: 后端报错 "Module not found"

**症状**: 后端启动失败，提示找不到模块

**解决方案**:
```bash
# 确保在正确的目录
cd intelligent_project_analyzer

# 重新安装依赖
pip install -r requirements.txt

# 重启服务
python -m uvicorn main:app --reload
```

---

### 问题3: 前端报错 "Cannot read property"

**症状**: 前端控制台显示类型错误

**排查步骤**:
1. 打开浏览器开发者工具（F12）
2. 查看Console标签页的完整错误信息
3. 检查Network标签页的API响应

**解决方案**:
```bash
# 重新构建前端
cd frontend-nextjs
npm run build
npm run dev
```

---

### 问题4: 交付物质量验证失败

**症状**: 后端日志显示"交付物质量不达标"

**这是正常的**:
- 系统会自动验证交付物质量
- 评分 >= 70 为合格
- 评分 < 70 会有警告，但不影响输出

**查看详情**:
```
⚠️ [v7.302] 交付物质量不达标 | 评分=65.0 | 问题数=3
  - 交付物3过于简短：'空间规划'
  - 交付物5缺少动词：'材质选择'
```

## 📊 性能监控

### 正常响应时间

| 阶段 | 预期时间 | 说明 |
|------|---------|------|
| 分析阶段 | 30-60秒 | LLM生成4条使命 |
| 前端渲染 | < 1秒 | 显示4条使命UI |
| 复制操作 | 即时 | 复制到剪贴板 |

### 性能指标

**正常范围**:
- L5锐度评分: 70-95
- 交付物数量: 10-15个
- 关键步骤: 5-8个
- 质量标准: 5-8项

**异常情况**:
- 响应时间 > 120秒 → 检查网络和LLM服务
- L5锐度 < 70 → 问题描述可能不够清晰
- 交付物 < 10个 → 可能是简单问题，正常
