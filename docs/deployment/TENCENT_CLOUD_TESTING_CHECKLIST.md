# 腾讯云上线测试检查表（可勾选版）

## 使用说明

- 适用阶段：开发阶段、稳定期、上线窗口、上线后运行。
- 使用方式：每次里程碑评审或发布前，逐项勾选并记录证据链接（CI 运行地址、测试报告、监控截图）。
- 放行原则：`P0` 条目必须全部通过；`P1` 条目需有负责人和补救计划。

---

## A. 开发阶段（持续进行）

### A1 静态与质量门禁（P0）

- [ ] Ruff 通过（后端与 tests 范围）
- [ ] `ci_doctor` 通过（workflow/阈值/文档一致性）
- [ ] 关键 API smoke 回归通过
- [ ] 无新增未白名单安全风险（Bandit/Safety 基线）

验收证据：

- CI 运行链接：
- `reports/ci-doctor-report.json`：
- 安全报告 artifact：

### A2 功能与回归（P0）

- [ ] 新功能具备对应单元测试
- [ ] 关键异常分支有测试覆盖
- [ ] 历史高频缺陷对应回归用例通过

### A3 不稳定测试治理（P1）

- [ ] flaky 用例已标记 `@pytest.mark.flaky`
- [ ] `docs/testing/FLAKY_TESTS.md` 已更新 owner/修复日期
- [ ] `flaky-isolated` 结果已复盘

## B. 稳定期（上线前冻结窗口）

### B1 集成与端到端（P0）

- [ ] 后端 + DB + Redis + 外部依赖联调通过
- [ ] 关键 E2E 路径通过（登录/提交/分析/报告）
- [ ] 失败重试、超时、降级路径验证通过

### B2 性能与容量（P0）

- [ ] 并发压测完成并达标（记录目标 TPS/QPS）
- [ ] P95/P99 延迟达标
- [ ] 长稳测试无明显内存泄漏/资源飙升

### B3 安全专项（P0）

- [ ] 鉴权与越权测试通过
- [ ] 注入/反序列化/敏感数据泄露检查通过
- [ ] 限流、防刷、告警策略生效

### B4 数据迁移与恢复（P0）

- [ ] 迁移脚本在预发布环境验证通过
- [ ] 回滚脚本可执行并验证成功
- [ ] 备份恢复演练成功（RTO/RPO 记录）

## C. 腾讯云生产部署阶段（上线窗口）

### C1 部署前（P0）

- [ ] 生产配置核对（ENV/密钥/证书/域名/白名单）
- [ ] 基础设施状态正常（CVM/CLB/Redis/MySQL/对象存储）
- [ ] 发布包版本可追溯（commit/tag/image digest）

### C2 部署后冒烟（P0）

- [ ] 健康检查接口正常
- [ ] 核心 API 可用，无 5xx
- [ ] 数据库与缓存连接正常
- [ ] 核心业务链路首单成功

### C3 灰度与放量（P0）

- [ ] 小流量灰度通过
- [ ] 错误率/延迟/资源指标稳定
- [ ] 放量策略执行并记录关键时间点

### C4 回滚准备（P0）

- [ ] 一键回滚方案可用
- [ ] 回滚责任人与执行口令明确
- [ ] 回滚触发阈值明确（错误率/延迟/SLO）
