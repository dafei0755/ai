复杂度0.883 > 0.45 → 推荐40+个任务

## 🎯 修复目标

**核心原则**: Few-shot示范案例的任务数量必须与复杂度推荐范围一致，否则会导致LLM学习错误的任务拆分模式。

**修复方案**:
1. **案例1（蛇口菜市场）**: 14个 → **38个任务**
2. **案例2（自闭症家庭住宅）**: 13个 → **36个任务**
3. **梯度覆盖**: 与狮岭村32个任务形成简单→中等→高复杂度梯度

## ✅ 质量保证

### 扩展原则

1. **粒度一致性**: 新增任务保持与原有任务相同的粒度标准
   - ❌ 避免"多个材料""多种技术"等混合表述
   - ✅ 每个材料/技术/功能独立成任务

2. **格式规范性**: 严格遵循"动词 + 对象 + 维度清单"格式
   - 研究类：搜索/调研/查找
   - 分析类：分析/对比/评估
   - 设计类：设计/制定/构建

3. **维度明确性**: 每个任务罗列3-5个具体调研维度
   - ❌ "深度研究XX"
   - ✅ "搜索 XX的 维度1、维度2、维度3、维度4"

4. **系统性完整性**: 保持原有调研逻辑的完整性
   - 标杆对标 → 深度调研 → 系统拆分 → 整合输出

5. **类型比例平衡**: 维持research为主导（75-80%）
   - Research: 75-80%
   - Analysis: 15-20%
   - Output: 5-10%

### 验证结果

✅ **粒度细分充分**: 将原有粗粒度任务拆分为独立任务（如材料策略 → 地面/墙面/天花/金属/玻璃各1个任务）

✅ **维度全面覆盖**: 从标杆学习到政策法规到用户体验，无遗漏

✅ **格式规范统一**: 所有新增任务严格遵循命名规范

✅ **类型比例合理**: 维持商业/功能项目的研究类任务主导地位
