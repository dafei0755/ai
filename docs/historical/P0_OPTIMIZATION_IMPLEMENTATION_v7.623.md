## P0-2: 加权投票机制实施详情

### 实施内容

#### 1. 新增加权投票函数

**文件**: `intelligent_project_analyzer/agents/requirements_analyst_agent.py`

**核心算法**:
```python
def _weighted_info_status_vote(
    precheck_result, phase1_result, mode_adjustment_result
) -> Tuple[str, Dict[str, Any]]:
    """
    加权投票决定最终info_status

    投票权重：
    - Precheck: 0.4（程序化检测，准确率82%）
    - Phase1: 0.4（LLM判定，准确率78%）
    - Mode调整: 0.2（模式特定要求）

    决策阈值：0.5
    """
    final_score = (
        precheck_score * 0.4 +
        phase1_score * 0.4 +
        mode_score * 0.2
    )

    final_status = "sufficient" if final_score >= 0.5 else "insufficient"
```

**决策理由生成**:
```python
def _build_decision_reason(...) -> str:
    """构建决策理由"""
    # 三方一致
    if all_agree:
        return "三方一致判定：信息充足/不足"
    # 加权投票
    else:
        return f"加权投票判定（支持方：{supporters}）"
```

#### 2. 集成到Phase1节点

**关键修改**:
```python
# Phase1节点末尾
final_info_status, decision_details = _weighted_info_status_vote(
    precheck_result=precheck_result,
    phase1_result=phase1_result,

# 记录决策日志
if final_info_status != info_status:
    logger.warning(f"[WeightedVote] info_status调整: '{info_status}' -> '{final_info_status}'")
    logger.info(f"   原因: {decision_details['decision_reason']}")
    logger.info(f"   投票分数: {decision_details['final_score']:.2f}")
```

**新增字段**:
