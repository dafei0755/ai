## 🔬 模块1: 需求分析 (v7.960增强版)

### 1.1 五层分析架构 (Phase2)

**文件**: `intelligent_project_analyzer/agents/requirements_analyst.py`

#### L1 事实层 (Facts Layer)
- **功能**: 提取客观物理信息
- **输出**: 项目类型、面积、位置、预算、时间节点
- **示例**: `"350㎡独栋别墅，上海郊区，800万预算"`

#### L2 用户模型层 (User Model Layer)
- **维度**: 心理模型 + 社会学特征 + 审美倾向
- **方法**: 关键词推理 + LLM深度分析
- **示例**:
  ```json
  {
    "psychological": "独立性强、追求掌控感、完美主义",
    "sociological": "高净值家庭、重视隐私、社交选择性",
    "aesthetic": "现代极简、东方禅意、克制优雅"
  }
  ```

#### L3 核心张力层 (Core Tension Layer)
- **功能**: 识别设计矛盾与平衡点
- **模式**: "X vs Y" (如"文化深度 vs 商业高端")
- **作用**: 指导后续维度选择和任务生成

#### L4 项目任务层 (Project Task Layer)
- **功能**: 精炼200字核心任务描述
- **原则**: 具象化、可执行、去抽象词汇
- **示例**: `"为XX打造融合书法符号的精品酒店，通过光影/材料转译传统文化"`

#### L5 锐度评分层 (Sharpness Layer)
- **功能**: 评估需求明确度 (0-100分)
- **用途**: 决定是否跳过问卷直达深度分析
- **阈值**: ≥80分可跳过 (当前系统34%场景达标)

#### 算法逻辑

```python
# 1. 文本合并
combined_text = project_task + character_narrative + physical_context + ...

# 2. 关键词匹配 (复用MotivationEngine动机类型)
for motivation_type in all_types:
    score = 0.0
    for keyword, weight in motivation_type.keywords.items():
        count = combined_text.count(keyword)
        score += weight * log(1 + count)  # 对数平滑,避免单个关键词过度影响

# 3. Sigmoid归一化到 0-1
raw_score = score / 5.0
normalized_score = 1 / (1 + exp(-raw_score))
normalized_score = (normalized_score - 0.5) * 2  # 偏移到0中心
normalized_score = clamp(0, 1, normalized_score)
```

**输入**: "融合传统书法文化的高端精品酒店设计，800万预算"

**输出**:

```json
  "aesthetic": 0.76,   // 精品酒店、设计氛围 → 高分
  "commercial": 0.72,  // 高端、预算明确 → 中高分
  "wellness": 0.35,    // 无明确健康需求 → 低分
  "technical": 0.28,   // 无智能化需求 → 低分

```

#### 降级保障

- **全0风险**: 若所有维度 < 0.3,强制设置 `functional=0.5`
- **异常处理**: 计算失败时使用12维默认向量 (functional=0.5, others=0.1-0.3)

### 2.4 智能融合算法

**函数**: `_merge_hybrid_tasks()` (L1475-1560)

#### Step 1: 规则任务强制保留

```python
merged.extend(rule_tasks)  # 5个规则任务全部保留
logger.info(f"✅ [Merge] 保留 {len(rule_tasks)} 个规则强制任务")
```

#### Step 2: LLM任务去重

**相似度计算** (`_calculate_task_similarity()`):

```python
# 混合指标
title_sim = SequenceMatcher(None, title1, title2).ratio()  # difflib字符串相似度

keywords1 = set(task1['source_keywords'])
keywords2 = set(task2['source_keywords'])
keyword_sim = len(keywords1 & keywords2) / len(keywords1 | keywords2)

similarity = 0.6 * title_sim + 0.4 * keyword_sim  # 加权平均
```

**去重逻辑**:

```python
for llm_task in llm_tasks:
    is_duplicate = False
    for rule_task in rule_tasks:
        similarity = _calculate_task_similarity(llm_task, rule_task)
        if similarity > 0.70:  # 阈值: 0.70 (v7.995调优)
            is_duplicate = True
            break

    if not is_duplicate:
        merged.append(llm_task)  # 保留独有任务
```

**示例**:
- `llm_1: "文化背景与传统元素提炼"` vs `rule_1: "文化背景与元素提炼"`
- 相似度: 0.81 → **去重** ✅
- `llm_2: "叙事节奏构建"` vs `rule_1: "文化背景与元素提炼"`
- 相似度: 0.12 → **保留** ✅

#### Step 3: 优先级排序

```python
def get_priority_score(task):
    priority = task.get('priority', 'medium')
    source = task.get('source', '')

    # 基础分: high=3, medium=2, low=1
    base_score = {'high': 3, 'medium': 2, 'low': 1}[priority]

    # 规则任务加权 +0.5
    bonus = 0.5 if source.startswith('rule_') else 0.0

    return base_score + bonus

merged.sort(key=get_priority_score, reverse=True)
```

**效果**: 规则任务优先出现,保障核心任务在前列
