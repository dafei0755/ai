**位置**: `core_task_decomposer.py` 第1983-1993行和第2040-2056行

**问题代码**:
```python
outline_prompt = f"""
输出JSON格式：
```json
{{
  "outline": [
    {{"category": "文化调研与保护", "subtask_count": 3, "priority": "high"}},
    ...
  ]
}}
```
"""
```

**问题分析**:
1. Python f-string中，`{{` 是转义语法，输出为单个 `{`
2. 但LLM看到prompt中的 `{{` 示例后，可能**模仿双花括号格式**
3. 导致LLM输出 `{{"category": ...}}`，JSON解析失败
4. 解析失败 → 回退策略 → 只生成6个简单任务

**影响范围**:
- Phase 1: 任务大纲生成（第1983-1993行）
- Phase 2: 详细任务生成（第2040-2056行）

## 技术细节

### Python f-string转义规则

```python
# 错误示例（LLM会模仿双花括号）
f"""
{{
  "key": "value"
}}
"""
# 输出: { "key": "value" }
# LLM看到: {{ "key": "value" }}
# LLM模仿: {{"key": "value"}}  ❌ JSON解析失败

# 正确示例（添加警告）
f"""
{{
  "key": "value"
}}

⚠️ 重要：JSON中必须使用单层花括号 {{ }}，不要使用双层花括号 {{{{ }}}}
"""
# LLM理解: 应该输出单层花括号
# LLM输出: {"key": "value"}  ✅ JSON解析成功
```

### 为什么会出现这个问题？

1. **YAML文件损坏**: 编辑器自动添加BOM，或误操作导致双花括号
2. **LLM模仿行为**: LLM看到示例中的 `{{`，认为这是正确格式
3. **解析失败**: `json.loads()` 无法解析 `{{"key": ...}}`
4. **回退策略**: 系统降级到简单规则提取，只生成6个基础任务
