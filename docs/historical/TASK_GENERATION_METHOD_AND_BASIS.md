# 任务梳理生成方法与依据详解

## 一、生成方法概述

任务梳理采用**混合生成策略（Hybrid Generation）**，结合了LLM智能生成和规则引擎补充。

### 核心流程（5步）

```
用户输入
  ↓
1. 复杂度分析（TaskComplexityAnalyzer）
  ↓
2. Track A: LLM两阶段生成（Phase 1大纲 + Phase 2详细任务）
  ↓
3. Track B: 规则引擎生成（关键强制任务）
  ↓
4. 智能融合（去重 + 优先级排序）
  ↓
5. 质量验证 + 动机标注
  ↓
输出：28-52个结构化任务
```

## 二、复杂度分析依据（Step 1）

### 评分维度（6个维度）

**代码位置**: `core_task_decomposer.py:32-172`

#### 1. 输入长度（权重15%）
```python
< 50字符   → 0.2分（简短）
50-150字符  → 0.4分（中等）
150-300字符 → 0.6分（详细）
> 300字符   → 0.8分（非常详细）
```

#### 2. 信息维度数量（权重40%，最关键）
检测13个维度：
- 项目类型（别墅、住宅、办公、商业、民宿等）
- 预算约束（数字+万/元、预算、成本等）
- 对标案例（对标、参考、案例等）
- 文化要素（文化、传统、在地、历史等）
- 客群分析（年龄、客群、用户等）
- 核心张力（引号对立、vs、矛盾等）
- 品牌主题（英文品牌名、主题等）
- 设计风格（风格、杂志级、高级感等）
- 功能需求（功能、需求、使用、场景等）
- 时间约束（月/年、工期、交付等）
- 地理位置（城市名、地点等）
- 特殊场景（老房翻新、旧改、文保等）
- 多业态融合（综合体、复合、一体化等）

**评分**: 包含维度数 / 10 × 0.40

**重要**: 面积（㎡/平米）**不影响**复杂度评分

#### 3. 结构化数据深度（权重25%）
检查6个关键字段：
- design_challenge（设计挑战）
- character_narrative（角色叙事）
- physical_context（物理环境）
- analysis_layers（分析层次）
- project_type（项目类型）
- core_goals（核心目标）

**评分**: 填充字段数 / 6 × 0.25

#### 4. 特殊场景识别（权重15%）
检测4种特殊场景：
- 诗意表达（如/似/若/像...般/样、意象、隐喻）
- 多元对立（3个以上引号概念、兼顾...与...与）
- 跨界融合（融合、结合、跨界、混搭）
- 文化深度（文化...文化、传统...现代）

**评分**: 检测到的场景数 / 3 × 0.15

#### 5. 句子数量（权重5-10%）
```python
≥ 5个句子/段落 → +0.10分
< 5个句子/段落 → +0.05分
```

#### 6. 具体数字（权重10%）
```python
≥ 3个数字 → +0.10分
```

### 任务数量映射（v7.998优化）

```python
复杂度 0-0.12   → 8-13个任务（简单需求）
复杂度 0.12-0.25 → 18-23个任务（中等需求）
复杂度 0.25-0.45 → 28-36个任务（复杂需求）
复杂度 0.45-1.0  → 40-52个任务（超大型项目）
```

## 三、LLM两阶段生成（Step 2）

### Phase 1: 生成任务大纲

**代码位置**: `core_task_decomposer.py:1952-2001`

**目标**: 生成8-10个标准任务类别

**提示词要求**:
1. 类别名称简洁通用（如"场地调研与分析"）
2. 避免过度具体化（❌"狮岭村农耕文化与传统工艺挖掘"）
3. 每个类别指定灵活的子任务数量（2-5个）
4. 按设计流程逻辑排序

**标准类别参考（8个）**:
```
1. 场地调研与分析（4-5个任务）
2. 文化背景与元素提炼（4-5个任务）
3. 用户需求与行为研究（3-4个任务）
4. 设计对标与参考研究（3-4个任务）
5. 概念设计与空间策略（4-5个任务）
6. 技术系统与材料策略（3-4个任务）
7. 商业模式与运营策略（2-3个任务）
8. 可持续性与创新策略（2-3个任务）
```

**输出格式**:
```json
{
  "outline": [
    {
      "category": "场地调研与分析",
      "subtask_count": 4,
      "priority": "high",
      "description": "针对狮岭村的地形、气候、景观等进行系统调研"
    }
  ],
  "total_tasks": 28-36
}
```

### Phase 2: 按类别生成详细任务

**代码位置**: `core_task_decomposer.py:2062-2130`

**目标**: 为每个类别生成2-5个具体任务

**提示词要求**:
1. 任务标题必须体现项目具体信息
2. 任务描述必须包含可执行的具体行动
3. 任务应是类别的具体子任务，避免重复类别名
4. 每个任务添加category字段

**示例对比**:
```
类别: "场地调研与分析"

❌ 错误: "场地调研与分析" → "对项目场地进行全面调研..."
✅ 正确: "狮岭村地形地貌与自然景观调研" → "调研狮岭村的地形高差、水系分布、植被类型，分析对建筑布局的影响..."
```

**输出格式**:
```json
{
  "tasks": [
    {
      "id": "task_1_1",
      "title": "狮岭村地形地貌与自然景观调研",
      "description": "调研狮岭村的地形高差、水系分布、植被类型...",
      "category": "场地调研与分析",
      "source_keywords": ["地形", "景观", "狮岭村"],
      "task_type": "research",
      "priority": "high",
      "execution_order": 1,
      "support_search": true
    }
  ]
}
```

### Few-shot学习增强

**代码位置**: `core_task_decomposer.py:1901-1931`

系统会根据项目类型和特征向量，自动选择最匹配的Few-shot示例：
- `commercial_dominant_01`: 商业主导型项目
- `functional_dominant_01`: 功能主导型项目
- `examples_registry`: 通用示例库

示例会注入到提示词中，提供任务拆解的粒度、格式、空间语言参考。

## 四、规则引擎生成（Step 3）

**代码位置**: `core_task_decomposer.py:1725-1803`

### 生成策略

1. **特征驱动**: project_features高分特征（≥0.7）→ 强制生成任务
2. **结构化提取**: design_challenge, character_narrative
3. **关键词匹配**: 对标、文化、客群等（优先级medium）

### 任务数量

```python
rule_task_count = max(3, int(recommended_max * 0.35))
```

例如：推荐36个任务 → 规则生成最多12个

### 特征映射示例

```python
project_features = {
    'cultural': 0.88,  # 文化认同高分
    'aesthetic': 0.76  # 美学体验高分
}

→ 自动生成:
- "文化背景与元素提炼"任务（强制）
- "美学风格与氛围营造"任务（强制）
```

## 五、智能融合（Step 4）

**代码位置**: `core_task_decomposer.py:1650-1722`

### 融合策略

1. **规则任务全部保留**（关键任务保障）
2. **LLM任务去重**（相似度>0.70的任务被过滤）
3. **优先级排序**（high > medium > low，规则来源+0.5分）
4. **数量截断**（如果超过max_tasks，智能截断）

## 六、质量验证（Step 5）

**代码位置**: `core_task_decomposer.py:2132-2173`（v7.999.10新增）

### 验证内容

1. **类别数量检查**
   - 期望: 8-10个
   - 警告: <6个或>12个

2. **类别任务数分布检查**
   - 期望: 每个类别2-5个任务
   - 警告: <2个或>6个

3. **输出日志**
```
✅ 任务质量验证通过: 9个类别，分布均衡

或

⚠️ 任务质量问题 (3个):

  - 类别'场地调研'任务过少: 1个（建议2-5个）
  - 类别'文化分析'任务过多: 7个（建议2-5个）
```

## 九、关键设计原则

### 1. 层次清晰
- 类别名称简洁通用（6-10字）
- 任务标题具体化（包含项目特定信息）
- 避免类别和任务名称重复

### 2. 粒度适中
- 每个类别2-5个任务（灵活调整）
- 重要类别（文化、场地）4-5个任务
- 简单类别（商业、可持续）2-3个任务

### 3. 逻辑有序
- 按设计流程排序：调研分析 → 策略定位 → 概念设计 → 技术实施
- 类别之间有清晰的逻辑关系

### 4. 可执行性
- 任务描述包含具体行动（调研什么、分析什么、设计什么）
- 避免模糊表述（❌"研究XX的设计理念"）
- 明确罗列调研维度（✅"建筑哲学、材料策略、空间理念、代表作品"）

### 5. 质量保障
- 自动质量验证机制
- Few-shot学习增强
- 混合生成策略（LLM + 规则）
- 智能去重和融合
