# 📊 需求分析师质量测试计划

## 🎯 测试目标

基于 sf/q.txt 中的复杂场景，评估 Requirements Analyst Agent 的：
1. **场景理解能力** - 能否准确识别用户需求
2. **信息充足性判断** - 能否正确判断是否需要问卷
3. **交付物识别** - 能否识别正确的交付物类型
4. **动机分型准确性** - 能否准确判断用户动机类型
5. **Fallback稳定性** - 降级机制是否可靠

## 🔍 测试方法

### 测试模式
- **当前状态**: Phase1/Phase2 使用 fallback（emoji error导致）
- **测试重点**: Fallback机制的质量和稳定性
- **评估标准**: 输出的合理性和一致性

### 测试流程
1. 读取 q.txt 中的场景
2. 逐个执行 Agent 分析
3. 记录结果（info_status, deliverables, recommended_next）
4. 分析 fallback 输出质量
5. 生成测试报告

## 📋 测试场景预览

从 q.txt 中选取代表性场景进行测试...

### 场景分类
- **简单明确** - 信息充足，目标清晰
- **中等复杂** - 部分信息缺失，需要推断
- **高度复杂** - 多重需求，信息模糊
- **边界情况** - 极端、特殊、罕见场景
