### 3.1 V7升级：社会关系建模+心理洞察专家 ✅

#### 核心改造
```yaml
版本: v1.0 → v2.0
名称: 情感洞察专家 → 社会关系与心理洞察专家
定位: 实验性角色 → 正式核心角色
触发率: 低 → 在多代同堂/再婚/合租/养老项目中成为核心角色
```

#### 新增主能力：社会关系建模 (A9)
1. **权力距离建模** (Power Distance Modeling)
   - 理论基础：Hofstede文化维度理论
   - 空间应用：高权力距离 vs 低权力距离 vs 混合型
   - 输出：中轴对称 vs 去中心化 vs 仪式/日常分离

2. **隐私分级系统** (Privacy Hierarchy)
   - 理论基础：Altman隐私调节理论、领域性理论
   - 四级分类：公共区 → 半公共区 → 半私密区 → 完全私密区
   - 核心检测：隐私冲突路径、视线穿透、心理庇护所

3. **冲突缓冲结构** (Conflict Buffer)
   - 理论基础：Pruitt & Rubin冲突管理理论
   - 四种缓冲：物理缓冲（灰空间）、时间缓冲（错峰）、功能缓冲（多路径）、视线缓冲
   - 高发场景：多代同堂、再婚家庭、合租、特殊需求

4. **关系演化适配** (Relationship Evolution)
   - 理论基础：家庭生命周期理论
   - 三大系统：可变房间、弹性边界、独立套间
   - 典型演化：儿童期→青春期、夫妻期→三代同堂、父母期→空巢期

5. **代际关系空间** (Intergenerational Space)
   - 理论基础：Vischer环境质量模型
   - 三类空间：连接空间（促进互动）、独立空间（保证自主）、缓冲空间（避免过密）

6. **社交距离控制** (Social Distance Control)
   - 理论基础：Hall近体学理论
   - 四级距离：亲密(0-45cm) → 个人(45-120cm) → 社交(120-360cm) → 公共(>360cm)

#### 保留辅能力：心理洞察 (原V7)
- 马斯洛需求层次、依恋理论、创伤知情设计、情绪地图、仪式行为、记忆锚点

#### 输出扩展
```json

    "evolution_adaptability": "关系演化适配策略",

  }

保留原有5个心理字段：
- emotional_landscape
- spiritual_aspirations
- psychological_safety_needs
- ritual_behaviors
- memory_anchors
```

### 3.2 V6-1增强：环境适应维度 🔄

#### 问题诊断
当前V6-1"结构与幕墙工程师"：
- ✅ 有能力：结构抗性（抗震、抗风、抗雪载）
- ❌ 缺能力：环境适应（气候策略、能源自持、舒适曲线）
- 🎯 对应缺口：A10 环境适应 (Environmental Adaptation)

#### 解决方案：模式注入（M8极端环境型设计）
在V6-1的system_prompt中，当检测到M8极端环境模式时，注入：

```yaml
   - 温度控制（极寒/极热环境的梯度过渡）
   - 湿度控制（防潮/除湿/加湿）
   - 氧气浓度（高海拔供氧）
   - 气压控制（地下/高海拔）
```

### 3.3 V6新增：6-5 灯光与视觉系统工程师 ⏳

#### 缺口诊断
- **当前状态**：灯光设计分散在V2（美学定灯光概念）和V6-2（机电配电系统）
- **系统缺口**：无人负责"灯光系统设计"本身
  - ❌ 缺灯光层次（Lighting Hierarchy）
  - ❌ 缺光线强化策略（Illumination Strategy）
  - ❌ 缺昼夜节律照明（Circadian Lighting）
  - ❌ 缺灯光与空间叙事的联动（Spatial Narrative Lighting）
- **对应缺口**：A5 灯光系统 (Lighting Architecture)

#### 角色定位
```yaml
"6-5":
  name: "灯光与视觉系统工程师"
  description: "作为V6首席实现官，聚焦于建筑的'第二结构'，负责灯光系统设计、视觉舒适度、昼夜节律照明及与空间叙事的联动。"
  keywords: ["灯光", "照明", "光线", "视觉", "昼夜节律", "情绪照明", "层次", "氛围"]
```

#### 核心理论框架
1. **灯光层次理论 (Lighting Hierarchy)**
   - 环境光（Ambient Lighting）：整体空间基础照度
   - 任务光（Task Lighting）：功能性照明（阅读、工作）
   - 重点光（Accent Lighting）：视觉焦点、艺术品、建筑特征
   - 装饰光（Decorative Lighting）：灯具本身的美学价值

2. **光线强化策略 (Illumination Strategy)**
   - 概念与光线的联动：用光线翻译设计概念
   - 空间序列的光线节奏：入口（暗）→ 过渡（渐亮）→ 高潮（强光）→ 沉静（柔光）
   - 材料与光线的互动：反射、吸收、透射、漫射

3. **昼夜节律照明 (Circadian Lighting)**
   - 理论基础：人体生物钟与光照强度、色温的关系
   - 动态色温：早晨（4000K-5000K偏冷白）→ 中午（5000K-6500K自然光）→ 傍晚（2700K-3000K暖黄）
   - 应用场景：养老院、医疗空间、长时间工作环境

4. **视觉舒适度 (Visual Comfort)**
   - 眩光控制（UGR统一眩光值<19）
   - 光照均匀性（照度均匀度 min/avg > 0.7）
   - 显色指数（CRI>80，高端空间CRI>90）

5. **情绪照明 (Emotional Lighting)**
   - 高照度+高色温 = 警觉、兴奋、工作状态
   - 低照度+低色温 = 放松、亲密、休息状态
   - 动态灯光场景：会议模式、休息模式、晚宴模式

### 3.4 Batch Scheduler正式化：V7纳入依赖链 ⏳

#### 问题诊断
当前 `batch_scheduler.py` 中：
```python
self.base_dependencies = {
    "V4": [],                    # 无依赖，第1批
    "V5": ["V4"],                # 依赖 V4，第2批
    "V3": ["V4", "V5"],          # 依赖 V4+V5，第3批
    "V2": ["V3", "V4", "V5"],    # 依赖全部，第4批
    "V6": ["V2"]                 # 依赖 V2，第5批
    # ❌ V7不在依赖链中，触发率低
}
```

#### 修改方案
```python
self.base_dependencies = {
    "V4": [],                    # 无依赖，第1批
    "V5": ["V4"],                # 依赖 V4，第2批
    "V3": ["V4", "V5"],          # 依赖 V4+V5，第3批
    "V7": ["V3", "V5"],          # 🆕 依赖 V3(叙事)+V5(场景)，第3批后
    "V2": ["V3", "V4", "V5", "V7"], # 依赖全部，第4批
    "V6": ["V2"]                 # 依赖 V2，第5批
}
```

#### 依赖逻辑
- **V7为何依赖V3？** V3提供人物叙事 → V7基于人物关系构建社会结构
- **V7为何依赖V5？** V5提供场景描述 → V7基于场景需求设计社交距离/隐私分级
- **V2为何依赖V7？** V2需要V7的社会关系洞察作为设计输入之一

#### 批次顺序
```

第3批: V3 (叙事专家)
第3.5批: V7 (社会关系+心理洞察) ← 🆕插入
第4批: V2 (设计总监)
第5批: V6 (首席工程师，包含6-5灯光)
```

## 七、对比：未实施 vs 实施后

### 实施前（当前系统）
```
用户："我是再婚家庭，有我的孩子和前任的孩子，怎么设计空间避免冲突？"

系统响应：
- V5可能建议"独立卧室+共享活动空间"（功能性但缺乏社会学深度）
- V7（旧版）可能给"心理安全基地""情绪缓冲区"（心理学角度但不结构化）
- ❌ 缺少：权力距离建模、隐私分级、冲突缓冲路径、继子女空间策略
```

### 实施后（升级系统）
```
用户："我是再婚家庭，有我的孩子和前任的孩子，怎么设计空间避免冲突？"

系统响应：
- 模式识别：M9 社会结构型设计
- V7（新版）成为核心角色，输出：
  1. 权力距离建模：分析新家庭的权力重构（继父母 vs 亲生父母）
  2. 隐私分级系统：
     - 完全私密区：各自卧室（独立套卫生间）
     - 半私密区：各自孩子的活动区（可选择隔离）
     - 半公共区：中性地带（客厅、餐厅，无归属感）
     - 公共区：入口、门厅
  3. 冲突缓冲结构：
     - 物理缓冲：过渡走廊、灰空间、各自有独立出入口
     - 功能缓冲：双动线系统（孩子可以不见面就回房）
     - 视线缓冲：卧室相互不可视，避免监视感
  4. 关系演化适配：
     - 可变边界：青春期后可转换为独立套间
     - 独立套间系统：为成年继子女预留"半独立"居住模块
  5. 前任记忆处理：
     - 新家不复制前任家庭的空间模式
     - 避免"前任孩子房间更大"的不公平感知
- V5配合提供场景：共享晚餐的仪式空间、各自休闲的独立区域
- V6设计：中性地带的材料与色彩（不偏向任何一方）
```

**关键差异**：
- ❌ 实施前：给功能方案，缺结构化社会学分析
- ✅ 实施后：给社会关系建模+空间策略，可直接指导设计
