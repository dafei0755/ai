# 任务复杂度测试用例集

## 测试目标
验证系统能够正确识别不同复杂度的任务，并匹配合适的专家组合。

---

- 字数限制明确

## 📊 统计分析

### 复杂度分布
- **极简（1-2专家）**: 5个案例（25%）
- **中等（3-4专家）**: 9个案例（45%）
- **复杂（5+专家）**: 6个案例（30%）

| 专家角色 | 出现次数 | 占比 |

| V2 项目总监 | 16次 | 80% |
| V3 诗意顾问 | 8次 | 40% |
| V4 文化专家 | 13次 | 65% |
| V5 接待动线 | 12次 | 60% |

### 关键判断特征

#### 简单任务特征
- ✅ 明确数量（"5个"、"10条"）
- ✅ 单一维度（只要颜色/材质/名字）
- ✅ 无空间设计
- ✅ 输入<100字

#### 中等任务特征
- ✅ 单一空间
- ✅ 有面积（50-300平米）
- ✅ 2-3个维度
- ✅ 有预算但<50万

#### 复杂任务特征
- ✅ 多个空间
- ✅ 大面积（>300平米）
- ✅ 多系统集成
- ✅ 预算>50万
- ✅ 技术复杂度高
- ✅ 标准化/连锁需求

## 🎯 实施建议

### 判断规则优先级
1. **明确排除规则**（最高优先级）
   - 数量词 + 单一任务 → 简单
   - 多空间 + 大面积 → 复杂

2. **面积阈值**
   - <50平米 + 单一功能 → 简单/中等
   - 50-300平米 → 中等
   - >300平米 → 复杂

3. **专家需求数量**
   - 需要1-2个专家 → 简单
   - 需要3-4个专家 → 中等
   - 需要5+个专家 → 复杂

4. **技术复杂度**
   - 无技术系统 → 简单
   - 基础技术（灯光/材质） → 中等
   - 多系统集成 → 复杂

### 特殊判断
- **文化深度任务**：即使简单，也可能需要V3+V4
- **跨界项目**：复杂度+1
- **情感化设计**：需要V2总监参与
- **纯技术咨询**：跳过文化专家，直接V9/V11
