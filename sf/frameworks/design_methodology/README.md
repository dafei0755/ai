# 设计方法论构建元框架 (Design Methodology)

> 适用场景：非项目级的元认知框架——如何构建、研究和传授设计方法论本身

## 框架概述

设计方法论构建元框架跳出具体项目，审视**设计方法论的方法论**。适用于事务所方法论
提炼、设计教育课程开发、行业标准体系构建等"关于设计的设计"场景。

## 五大核心维度

### 1. 研究对象定义 (Research Object Definition)
- **关键问题**：方法论研究的对象是什么——一类项目？一种技术？一个设计流程？
- **判断标准**：研究对象是否可被操作化定义，具有清晰的外延和内涵
- **常见误区**：研究对象过于宽泛——"如何做好室内设计"不是可研究的方法论对象

### 2. 知识提取结构 (Knowledge Extraction Structure)
- **关键问题**：如何从实践经验中提取可复用的知识结构？
- **判断标准**：提取的知识是否具备"可教授性"——另一个人能否依据此方法论复现设计过程
- **常见误区**：将个人直觉包装为方法论——"我觉得应该这样做"不构成方法论

### 3. 可操作化 (Operationalization)
- **关键问题**：方法论是否包含可执行的步骤/工具/模板？
- **判断标准**：是否提供了从输入→处理→输出的完整操作手册
- **常见误区**：方法论停留在理念层面，缺乏实操工具

### 4. 边界与局限 (Boundary and Limitation)
- **关键问题**：方法论的适用范围、前提条件和失效场景是什么？
- **判断标准**：是否诚实声明了"这个方法论不适用于什么情况"
- **常见误区**：将方法论宣称为万能——"适用于所有室内设计项目"

### 5. 方法论验证 (Methodology Validation)
- **关键问题**：方法论的有效性如何被验证和迭代？
- **判断标准**：是否设计了验证实验——在新项目上应用方法论并评估效果
- **常见误区**：方法论一经提出即被视为定论，不设置迭代机制

## 分析深度指南

| 层级 | 内容要求 |
|------|----------|
| L1 描述 | 界定方法论研究对象和目标受众 |
| L2 分析 | 梳理现有实践知识，识别可提取模式 |
| L3 策略 | 构建方法论框架（步骤+工具+模板） |
| L4 验证 | 设计验证实验和迭代机制 |
| L5 远见 | 方法论的传播策略和演化路径 |

## 知识注入要点

- 设计研究方法论：Archer、Cross、Schön 反思性实践
- 知识管理：Nonaka SECI 模型（隐性知识→显性知识的转化）
- 模式语言：Christopher Alexander 的 Pattern Language 方法
- 设计思维：Stanford d.school 的 Design Thinking 五阶段模型
- 中国语境：匠人传承中的"口传心授"如何系统化为可复用方法论
