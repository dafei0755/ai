# 跨学科转译方法论引擎 (Interdisciplinary Translation)

> 适用场景：将非设计学科（行为经济学/神经科学/社会学等）的理论体系转译为空间操作

## 框架概述

跨学科转译方法论引擎解决的核心问题是：**如何将其他学科的严谨理论转化为可操作的
空间设计策略**，同时保持学术诚信——承认转译过程中的信息损失和边界条件。

## 五大核心维度

### 1. 学科识别 (Discipline Identification)
- **关键问题**：项目需求触及了哪些非设计学科的知识域？
- **判断标准**：是否准确识别了相关学科，并引用了该学科的一手文献而非二手解读
- **常见误区**：泛泛引用"心理学"而不区分认知心理学、社会心理学、环境心理学

### 2. 核心理论提取 (Core Theory Extraction)
- **关键问题**：从相关学科中提取了哪些可空间化的核心理论？
- **判断标准**：提取的理论是否有明确的实验/实证基础，而非通俗化的观点
- **常见误区**：将科普读物中的"金句"当作理论依据

### 3. 空间操作映射 (Spatial Operation Mapping)
- **关键问题**：理论概念如何映射为具体的空间参数（尺度、材料、光线、声学等）？
- **判断标准**：映射关系是否逻辑可追溯——从理论到空间操作的推理链是否完整
- **常见误区**：跳跃式映射——直接从"行为经济学的锚定效应"跳到"入口设置价格标签"

### 4. 验证方法 (Validation Method)
- **关键问题**：转译后的空间策略如何验证其有效性？
- **判断标准**：是否提出了可执行的验证方案（用户测试、POE评估、A/B对比等）
- **常见误区**：假设转译自动有效，不设置验证环节

### 5. 边界声明 (Boundary Declaration)
- **关键问题**：转译的适用范围和局限性是什么？
- **判断标准**：是否诚实声明了转译过程中的信息损失、文化差异、样本局限
- **常见误区**：过度宣称转译的普适性——"神经科学证明 X 色彩让所有人放松"

## 分析深度指南

| 层级 | 内容要求 |
|------|----------|
| L1 描述 | 识别需求涉及的学科领域，列出候选理论 |
| L2 分析 | 提取核心理论，评估其空间化可行性 |
| L3 策略 | 建立理论→空间操作的映射方案 |
| L4 验证 | 设计验证方法，预设效果评估标准 |
| L5 远见 | 声明转译边界，提出持续优化路径 |

## 知识注入要点

- 转译方法论：Design Science（设计科学）、Evidence-Based Design（循证设计）
- 环境心理学：Kaplan 注意力恢复理论、Ulrich 压力缓解理论
- 行为经济学：Thaler & Sunstein "Nudge"（助推）理论在空间中的应用
- 神经建筑学：Academy of Neuroscience for Architecture (ANFA) 的研究范式
- 社会学：Lefebvre 空间生产理论、Goffman 拟剧论在空间行为中的应用
