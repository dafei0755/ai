# 多尺度统筹框架 (Multi-Scale Integration)

> 适用场景：城市→片区→建筑→景观→室内→家具的跨尺度协同决策项目

## 框架概述

多尺度统筹框架用于处理从宏观规划到微观细节的跨尺度设计项目。核心挑战在于
**不同尺度之间的设计语言传递**——如何确保城市层面的设计意图在建筑、景观、室内、
家具各层级上被忠实继承并合理变异。

## 五大核心维度

### 1. 尺度界定 (Scale Definition)
- **关键问题**：本项目涉及哪些尺度？每个尺度的主控方是谁？
- **判断标准**：能否清晰画出"尺度嵌套图"，标注每层的管控红线和自由度区间
- **常见误区**：把"多尺度"等同于"从大到小依次设计"——忽略了尺度间的反馈环路

### 2. 体量控制 (Volume Control)
- **关键问题**：建筑体量如何回应城市天际线、街道尺度、人体尺度？
- **判断标准**：是否给出了超越"容积率/建筑密度"的体量生成逻辑（如视线通廊、风环境、日照权）
- **常见误区**：仅用数字指标控制体量，缺乏空间体验层面的论证

### 3. 公共-私密梯度 (Public-Private Gradient)
- **关键问题**：从城市公共空间到建筑私密空间，过渡层级是否完整？
- **判断标准**：能否画出连续的"公共性衰减曲线"，识别灰空间节点
- **常见误区**：简单的公/私二分法，缺乏半公共/半私密的过渡设计

### 4. 灰空间体系 (Grey Space System)
- **关键问题**：连接不同尺度的过渡空间（廊、院、阳台、前室）如何系统化设计？
- **判断标准**：灰空间是否承载了社交、气候调节、视觉缓冲等多重功能
- **常见误区**：灰空间被当作"剩余空间"而非"设计空间"

### 5. 跨尺度一致性 (Cross-Scale Consistency)
- **关键问题**：设计语言（材料、色彩、比例、肌理）如何从大尺度传递到小尺度？
- **判断标准**：能否追溯室内细节的"城市基因"——从城市肌理到家具纹样的逻辑链
- **常见误区**：各尺度独立设计后强行统一风格

## 分析深度指南

| 层级 | 内容要求 |
|------|----------|
| L1 描述 | 识别项目涉及的尺度范围，标注关键接口 |
| L2 分析 | 分析不同尺度间的约束传递与自由度 |
| L3 策略 | 提出跨尺度设计语言传递策略 |
| L4 验证 | 选取关键接口进行跨尺度一致性验证 |
| L5 远见 | 预判未来扩建/改造对多尺度体系的影响 |

## 知识注入要点

在专家执行时，应注入以下领域知识：
- 城市设计理论：Kevin Lynch《城市意象》、Jan Gehl《交往与空间》
- 尺度理论：Christopher Alexander《建筑的永恒之道》的模式语言
- 景观都市主义：Charles Waldheim 的景观基础设施理论
- 中国语境：山水城市、传统村落的多尺度智慧
