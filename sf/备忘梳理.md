 ontology.yaml — 问"这个项目需要关注什么？"       │
│     对象：项目本身的属性空间                         │
│     输出：分析维度清单（注入prompt）                  │
│     粒度：12种业态 × 每业态3-4分类 × 每分类3-5维度    │
│     比喻：体检表（查什么项目）                        │
└──────────────┬───────────────────────────────────┘
               │ 注入到专家prompt
               ▼
┌──────────────────────────────────────────────────┐
│  ② 多角色系统 (roles/*.yaml) — 问"谁来做这个项目？" │
│     对象：执行者的身份与分工                         │
│     输出：选定专家组 + 任务分派                      │
│     粒度：V2设计总监 / V3叙事专家 / V4研究员 /       │
│           V5场景专家 / V6工程师 / V7情感专家          │
│     比喻：医院科室（哪个科的医生来看）                │
└──────────────┬───────────────────────────────────┘
               │ 每个角色声明自己拥有哪些能力
               ▼
┌──────────────────────────────────────────────────┐
│  ③ 12_Ability_Core — 问"设计师需要什么能力？"       │
│     对象：设计能力的知识体系                         │
│     输出：12种能力 × 每能力5层成熟度(L1→L5)          │
│     粒度：A1概念建构 / A2空间结构 / A3叙事节奏 /     │
│           A4材料 / A5灯光 / A6功能 / A7资本 /        │
│           A8技术 / A9社会 / A10环境 / A11运营 / A12文明│
│     比喻：技能树（医生的专业能力等级）                │
└──────────────────────────────────────────────────┘
               ↕ 能力如何组合运用？
┌──────────────────────────────────────────────────┐
│  ④ 10_Mode_Engine — 问"用什么方式做这个项目？"      │
│     对象：设计策略/思维模式                          │
│     输出：10种设计模式，每种模式调用不同能力组合       │
│     粒度：M1概念驱动 / M2功能效率 / M3情绪体验 /     │
│           M4资本资产 / M5乡建在地 / M6城市更新 /      │
│           M7技术整合 / M8极端环境 / M9社会结构 / M10未来│
│     比喻：治疗方案（中医/西医/手术/保守…）
